{{BOLD}}ergo quickstart{{RESET}}
{{CYAN}}Complete command reference for agents and humans.{{RESET}}

{{HEADER}}1. MODEL{{RESET}}

Ergo stores tasks.

  - A leaf holds work, state, dependencies, body, and results.
  - A container is a root task with children. It completes when every child is
    done or canceled; it has no lifecycle, claim, or result of its own.

The JSON placement field and list filter retain the compatibility name `epic`;
the model calls the parent a container.

Leaf states express intent:

  todo       open and unclaimed
  doing      open and claimed by one agent
  blocked    open with an identified impediment
  done       objective satisfied
  canceled   objective no longer wanted

A claim exists exactly while state is doing. Readiness is derived:

  todo + every dependency complete = ready

A todo task with unmet dependencies is waiting. Blocked means someone has
identified an impediment requiring action.

{{HEADER}}2. INITIALIZE AND INSPECT{{RESET}}

  {{CMD}}ergo init{{RESET}}
  {{CMD}}ergo init path/to/repo{{RESET}}
  {{CMD}}ergo where{{RESET}}

`init` creates `.ergo/plans.jsonl` and `.ergo/lock`. Discovery walks upward from
the current directory or from `--dir <path>`.

  {{CMD}}ergo list{{RESET}}
  {{CMD}}ergo list --ready{{RESET}}
  {{CMD}}ergo list --all{{RESET}}
  {{CMD}}ergo list --epic OFKSTE{{RESET}}
  {{CMD}}ergo --json list{{RESET}}
  {{CMD}}ergo --json show ABCDEF{{RESET}}

Default list omits done and canceled work. `--ready` shows claimable todo leaves.
`--all` includes closed work and conflicts with `--ready`. `--epic` selects one
container. Show returns one leaf or one container with its children.

{{HEADER}}3. CLAIM WORK{{RESET}}

Use a stable identity such as `model@hostname`.

Claim the oldest ready task:
  {{CMD}}ergo --json claim --agent sonnet@agent-host{{RESET}}

Claim or resume a chosen task:
  {{CMD}}ergo --json claim ABCDEF --agent sonnet@agent-host{{RESET}}

Automatic claim selects the oldest ready todo leaf. Specific claim accepts todo,
blocked, done, canceled, and legacy error. It bypasses readiness and may resume
work with incomplete dependencies; use it only for a deliberately chosen task.

Claim assigns identity and doing atomically. Repeating it as the owner is a
no-op; another agent gets a conflict. Containers cannot be claimed.

Claim JSON supplies exact exits for the claimed task:

  "next_commands": {
    "done": "ergo --json done ABCDEF",
    "block": "ergo --json block ABCDEF",
    "cancel": "ergo --json cancel ABCDEF",
    "release": "ergo --json release ABCDEF"
  }

No ready work also exits 0:
  {"status":"no_ready","message":"No ready ergo tasks."}

{{HEADER}}4. FINISH OR CLASSIFY WORK{{RESET}}

Choose the command that states what is now true:

  {{CMD}}ergo done ABCDEF{{RESET}}       objective satisfied
  {{CMD}}ergo block ABCDEF{{RESET}}      identified impediment requires action
  {{CMD}}ergo cancel ABCDEF{{RESET}}     objective no longer wanted
  {{CMD}}ergo release ABCDEF{{RESET}}    unfinished work is ready for another attempt

Done, block, and cancel accept any readable leaf state. Release accepts todo,
doing, blocked, or legacy error and establishes unclaimed todo. It rejects done
and canceled; specific claim resumes closed work. Every exit clears the claim.

Repeating an established postcondition adds no state event, though it may still
replace the body or attach a result. There is no `set` or `reopen` command.

{{HEADER}}5. BODY NOTES AND RESULTS{{RESET}}

Feed longer bodies from a file, never as JSON or a multiline shell argument:

  POSIX or cmd.exe:  {{CMD}}ergo done ABCDEF < completion.md{{RESET}}
  PowerShell:        {{CMD}}Get-Content -Raw completion.md | ergo done ABCDEF{{RESET}}

The later `printf` example is POSIX-only and suited to short bodies.

Attach an existing project file:

  {{CMD}}ergo done ABCDEF --result docs/spec.md{{RESET}}
  {{CMD}}ergo block ABCDEF --result .scratch/repro.md --summary "Reproduction evidence"{{RESET}}
  {{CMD}}ergo release ABCDEF --result .scratch/attempt.md --summary "Retry evidence"{{RESET}}

Piped lifecycle stdin replaces the body literally in the same locked batch as
result, state, and claim changes. TTY stdin leaves it unchanged.

`--result` must name an existing file inside the project and outside `.ergo/`.
`--summary` requires it; the path is the default summary. Ergo records SHA-256,
mtime, and git commit at attachment. Existing results remain as attempt history.

Attach a late result by repeating the current lifecycle command:

  {{CMD}}ergo done ABCDEF --result docs/followup.md{{RESET}}

{{HEADER}}6. CREATE AND ORGANIZE WORK{{RESET}}

  {{CMD}}ergo new task '{"title":"Login"}'{{RESET}}
  {{CMD}}printf '%s\n' '## Goal' '- Implement signup' | ergo new task '{"title":"Signup"}'{{RESET}}
  {{CMD}}ergo new task '{"title":"Session tokens","epic":"OFKSTE"}'{{RESET}}
  {{CMD}}ergo --json new task '{"title":"Fix CVE","claim":"sonnet@agent-host"}'{{RESET}}
  {{CMD}}ergo new task '{"title":"Document decision","state":"done"}'{{RESET}}

JSON accepts `title`, `epic`, `state`, `claim`, and `result`. Title is required;
unknown fields fail. State is todo, doing, blocked, done, or canceled. Claim
implies doing; doing otherwise requires `--agent`. New work cannot use legacy
error. Piped stdin is the literal body; an empty pipe creates an empty body.

Use focused commands for later edits:

  {{CMD}}ergo title ABCDEF "Clarify authentication failure"{{RESET}}
  {{CMD}}printf '%s\n' '## Goal' '- Clarify the failure' | ergo body ABCDEF{{RESET}}
  {{CMD}}printf '' | ergo body ABCDEF{{RESET}}
  {{CMD}}ergo move ABCDEF OFKSTE{{RESET}}
  {{CMD}}ergo move ABCDEF --root{{RESET}}

Title rejects blank text. Body requires a pipe; an empty pipe clears it. Both
work on leaves and containers.

Move accepts leaves only. The destination is a root container or an unclaimed
root todo task with no results; that task becomes a container on its first child.
Containers cannot move or nest. Self-moves and container-child dependency
conflicts fail. Moving to the current parent is a no-op.

Create a container and several children from one file:

  {{CMD}}ergo --json plan --file tasks.md '{"title":"Auth system"}'{{RESET}}

`tasks.md` contains chunks separated by a line that is exactly `---`:

  # Schema
  Create tables and indexes.
  ---
  # Endpoints
  Add signup and login handlers.

Each chunk begins with `# Title`; the rest is its body. File order creates no
dependencies.

{{HEADER}}7. DEPENDENCIES{{RESET}}

  {{CMD}}ergo sequence TASK_A TASK_B{{RESET}}
  {{CMD}}ergo sequence TASK_A TASK_B TASK_C{{RESET}}
  {{CMD}}ergo sequence rm TASK_A TASK_B{{RESET}}

`sequence A B` makes B depend on A. A longer sequence links each adjacent pair.
`sequence rm A B` removes the edge where B depends on A.

Self-dependencies, cycles, and dependencies between a container and its child
fail. A container dependency completes when all its children are done or
canceled.

{{HEADER}}8. OUTPUT, FLAGS, AND ERRORS{{RESET}}

Global flags may appear with any command:

  {{CMD}}ergo --json ...{{RESET}}              one machine-readable JSON value
  {{CMD}}ergo --agent model@host ...{{RESET}}  caller identity
  {{CMD}}ergo --dir path/to/repo ...{{RESET}}  graph discovery start
  {{CMD}}ergo --help{{RESET}}                  quick reference
  {{CMD}}ergo --version{{RESET}}               build version

Successful `--json` commands write one JSON value and no extra stdout. Direct
mutations return `kind`, `id`, `updated_fields`, and state. `updated_fields`
names real changes or appended results; claim cleanup is `claim`. A no-op returns
an empty array. `claimed_by` appears only when the result is doing.

Failures return nonzero with an actionable stderr message. A missing graph
suggests `ergo init` and `ergo --dir <path>`. Removed v1 forms point to v2.

{{HEADER}}9. MAINTENANCE AND LEGACY PLANS{{RESET}}

  {{CMD}}ergo prune{{RESET}}
  {{CMD}}ergo prune --yes{{RESET}}
  {{CMD}}ergo compact{{RESET}}
  {{CMD}}ergo version{{RESET}}

Prune previews deletion. `--yes` tombstones done and canceled leaves, then empty
containers. Compact rewrites the live graph and removes pruned history.

Ergo reads `.ergo/plans.jsonl` and legacy `.ergo/events.jsonl` without migration.
Opening never rewrites. Replay and compact preserve unresolved legacy error and
claimed-blocked tasks. A lifecycle command normalizes only its task; v2 can
replay events appended by older binaries.

Reads and writes acquire `.ergo/lock`. Mutations append a validated event batch
under that lock. Oldest-ready selection and claim are atomic.
