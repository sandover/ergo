{{BOLD}}ergo quickstart{{RESET}}
{{CYAN}}Complete reference manual. Optimized for agents and humans.{{RESET}}

{{HEADER}}1. INITIALIZE{{RESET}}

{{CMD}}ergo init{{RESET}}
Creates `.ergo/` in the current directory.

{{CMD}}ergo where{{RESET}}
Prints the active `.ergo/` path.

{{HEADER}}2. INSPECT WORK{{RESET}}

Human overview:
  {{CMD}}ergo list{{RESET}}

Machine inventory:
  {{CMD}}ergo --json list{{RESET}}

Only actionable work:
  {{CMD}}ergo --json list --ready{{RESET}}

Full detail for one task or container:
  {{CMD}}ergo --json show ABCDEF{{RESET}}

Short human summary for one task:
  {{CMD}}ergo show ABCDEF --short{{RESET}}

Use `show` before mutating when you need the body, child tasks, or dependencies.

{{HEADER}}3. AGENT LOOP{{RESET}}

Your identity should be `<model>@<hostname>`.

Claim oldest ready work:
  {{CMD}}ergo --json claim --agent sonnet@agent-host{{RESET}}

Claim a specific task:
  {{CMD}}ergo --json claim ABCDEF --agent sonnet@agent-host{{RESET}}

Claim inside one container:
  {{CMD}}ergo --json claim --agent sonnet@agent-host --epic ABCDEF{{RESET}}

Finish the task. Never leave it in `doing`.

Mark blocked:
  {{CMD}}ergo --json set ABCDEF '{"state":"blocked"}'{{RESET}}

Mark done and attach a result file:
  {{CMD}}ergo --json set ABCDEF '{"state":"done","result":"docs/spec.md"}'{{RESET}}

Claim and `doing` move together. Finishing (`done` or `canceled`) clears the claim. `error` retains it.

{{HEADER}}4. CREATE WORK{{RESET}}

`new task [json]` creates exactly one task. The JSON argument carries metadata. If stdin is piped, it becomes the body.

Create a small task:
  {{CMD}}ergo new task '{"title":"Login"}'{{RESET}}

Create a task inside a container:
  {{CMD}}ergo new task '{"title":"Session tokens","epic":"OFKSTE"}'{{RESET}}

Create a task with a multi-line body:
  {{CMD}}printf '%s\n' '## Goal' '- Implement signup' '' '## Acceptance Criteria' '- Password reset works' | ergo new task '{"title":"Login"}'{{RESET}}

Create and claim in one step:
  {{CMD}}ergo --json new task '{"title":"Fix CVE","claim":"sonnet@agent-host"}'{{RESET}}

Unknown keys are rejected. `new task` requires `title`.

{{HEADER}}5. UPDATE WORK{{RESET}}

`set <id> [json]` updates exactly one task. The JSON argument changes metadata. If stdin is piped, it replaces the body.

Rename or reparent:
  {{CMD}}ergo set ABCDEF '{"title":"Login API","epic":"OFKSTE"}'{{RESET}}

Replace the body with markdown from stdin:
  {{CMD}}printf '%s\n' '## Status' '- Waiting on review' | ergo set ABCDEF '{"state":"blocked"}'{{RESET}}

Clear the body:
  {{CMD}}printf '' | ergo set ABCDEF{{RESET}}

Attach another result later:
  {{CMD}}ergo set ABCDEF '{"result":"docs/followup.md"}'{{RESET}}

{{HEADER}}6. PLAN A FEATURE{{RESET}}

`plan --file <path> [json]` creates one container and child tasks from markdown.

Example command:
  {{CMD}}ergo --json plan --file tasks.md '{"title":"Auth system"}'{{RESET}}

Example file:
  {{CMD}}cat > tasks.md <<'EOF'
# Schema
Create the tables and indexes.
---
# Endpoints
Add signup and login handlers.
---
# Docs
Document the auth flow.
EOF{{RESET}}

Rules:
  - Split chunks on a line that is exactly `---`.
  - Each chunk must start with `# Title`.
  - The heading becomes the child title.
  - The remaining markdown becomes the child body.
  - The command JSON must include the top-level container `title`.
  - File order does not create dependencies.

Add dependencies explicitly after planning:
  {{CMD}}ergo sequence TASK_SCHEMA TASK_ENDPOINTS TASK_DOCS{{RESET}}

{{HEADER}}7. DEPENDENCIES{{RESET}}

Enforce order (A before B):
  {{CMD}}ergo sequence TASK_A TASK_B{{RESET}}

Chain order (A before B before C):
  {{CMD}}ergo sequence TASK_A TASK_B TASK_C{{RESET}}

Remove one edge:
  {{CMD}}ergo sequence rm TASK_A TASK_B{{RESET}}

Dependencies are allowed between any two non-ancestor tasks. Cycles are forbidden.

{{HEADER}}8. PRUNE + COMPACT{{RESET}}

Preview logical deletion:
  {{CMD}}ergo prune{{RESET}}

Apply logical deletion:
  {{CMD}}ergo prune --yes{{RESET}}

Physically rewrite the log after pruning:
  {{CMD}}ergo compact{{RESET}}

Prune removes `done` and `canceled` tasks, then removes any empty containers left behind.

{{HEADER}}9. CONTAINERS + RULES{{RESET}}

A task becomes a container when it has children.

Containers:
  - complete when all children are `done` or `canceled`
  - cannot hold state, claim, or results directly
  - cannot be nested more than one level deep

State set:
  - `todo`, `doing`, `blocked`, `error`, `done`, `canceled`

Invariants:
  - `doing` requires a claim
  - `error` requires a claim
  - `todo`, `done`, and `canceled` clear the claim
  - `blocked` may keep or drop the claim

Agent ID format:
  - `<model>@<hostname>` is recommended, not enforced
