{{BOLD}}ergo quickstart{{RESET}}
{{CYAN}}Complete guide for agents and humans.{{RESET}}

{{HEADER}}1. START HERE{{RESET}}

Ergo stores a task graph in the repository. This is the normal agent loop:

  {{CMD}}ergo list --ready{{RESET}}
  {{CMD}}ergo claim --agent model@host{{RESET}}

Claim prints the complete task. It begins with YAML front matter:

  ---
  id: "ABCDEF"
  title: "Add password login"
  state: "doing"
  claimed_by: "model@host"
  ---

The task body follows as Markdown. Dependencies, lifecycle messages, and result
links appear in named sections. The final `## Next` section prints exact commands
for `done`, `block`, `cancel`, and `release`. Read the ID from the fixed second
line and run the command that states the outcome.

No ready work is not an error. Automatic claim prints `No ready ergo tasks.` and
exits 0.

{{HEADER}}2. MODEL{{RESET}}

A leaf holds work. Its state is:

  todo       open and unclaimed
  doing      open and claimed by one agent
  blocked    open; an identified impediment requires action
  done       objective satisfied
  canceled   objective no longer wanted

A claim exists exactly while state is doing. Readiness is derived:

  todo + every dependency complete = ready

A todo task with unmet dependencies is waiting, not explicitly blocked.

A container is a root task with children. It completes when every child is done
or canceled. It has no lifecycle, claim, or result of its own. The creation field
and list filter use the compatibility name `epic` for a container.

{{HEADER}}3. INITIALIZE, FIND, AND INSPECT{{RESET}}

  {{CMD}}ergo init{{RESET}}
  {{CMD}}ergo init path/to/repo{{RESET}}
  {{CMD}}ergo where{{RESET}}

Init creates `.ergo/plans.jsonl` and `.ergo/lock`, then prints the `.ergo` path.
Discovery walks upward from the current directory or `--dir <path>`.

  {{CMD}}ergo list{{RESET}}                 active work; omits done and canceled
  {{CMD}}ergo list --ready{{RESET}}         claimable leaves
  {{CMD}}ergo list --all{{RESET}}           include closed work
  {{CMD}}ergo list --epic OFKSTE{{RESET}}   one container and its children
  {{CMD}}ergo show ABCDEF{{RESET}}          complete leaf or container

`--ready` and `--all` conflict. Lists label work as ready, waiting, doing,
blocked, done, canceled, or legacy error. Show uses the same complete Markdown
document as claim; a container document includes every child.

{{HEADER}}4. CREATE AND ORGANIZE{{RESET}}

Create a leaf. The generated six-character ID is the complete output:

  {{CMD}}ergo new task '{"title":"Add login"}'{{RESET}}
  {{CMD}}ergo new task '{"title":"Session tokens","epic":"OFKSTE"}'{{RESET}}
  {{CMD}}ergo new task '{"title":"Fix CVE","claim":"model@host"}'{{RESET}}

The optional inline JSON fields are `title`, `epic`, `state`, `claim`, and
`result`. Title is required and unknown fields fail. State is todo, doing,
blocked, done, or canceled. Claim implies doing; a doing state without a claim
uses global `--agent`. New work cannot use the legacy error state.

Pipe multiline Markdown into the initial body. Do not put it in JSON or a shell
argument:

  {{CMD}}printf '%s\n' '## Goal' '- Implement signup' | ergo new task '{"title":"Signup"}'{{RESET}}

An empty pipe creates an empty body. Without a pipe, the body is also empty.

Create a container and children from a file:

  {{CMD}}ergo plan --file tasks.md '{"title":"Authentication"}'{{RESET}}

`tasks.md` contains chunks separated by a line that is exactly `---`:

  # Schema
  Create tables and indexes.
  ---
  # Endpoints
  Add signup and login handlers.

Each chunk begins with `# Title`; the rest is its body. Plan prints the container
ID, every child ID and title, and the dependency count. File order creates no
dependencies.

Edit focused fields directly:

  {{CMD}}ergo title ABCDEF "Clarify authentication failure"{{RESET}}
  {{CMD}}printf '%s\n' '## Goal' '- Clarify the failure' | ergo body ABCDEF{{RESET}}
  {{CMD}}printf '' | ergo body ABCDEF{{RESET}}
  {{CMD}}ergo move ABCDEF OFKSTE{{RESET}}
  {{CMD}}ergo move ABCDEF --root{{RESET}}

Title rejects blank text. Body is the only existing-task body writer. It requires
piped stdin and treats it literally; an empty pipe clears the body. Both accept
leaves and containers.

Move accepts leaves only. A destination must be a root container or a clean,
unclaimed root todo task with no results; that task becomes a container on its
first child. Containers cannot move or nest. A same-parent move is a no-op.

{{HEADER}}5. CLAIM AND RESUME{{RESET}}

Use a stable identity such as `model@hostname`:

  {{CMD}}ergo claim --agent sonnet@agent-host{{RESET}}          oldest ready task
  {{CMD}}ergo claim ABCDEF --agent sonnet@agent-host{{RESET}}   chosen task

Automatic claim selects only the oldest ready todo leaf. Specific claim accepts
todo, blocked, done, canceled, and legacy error. It bypasses readiness, so use it
only when deliberately resuming that ID.

Claim assigns doing and the identity atomically. Repeating it as the owner is a
no-op. Another identity gets a conflict. Containers cannot be claimed.

{{HEADER}}6. FINISH, NOTE, OR RETRY{{RESET}}

Choose the command that states what is now true:

  {{CMD}}ergo done ABCDEF{{RESET}}       objective satisfied
  {{CMD}}ergo block ABCDEF{{RESET}}      impediment requires action
  {{CMD}}ergo cancel ABCDEF{{RESET}}     objective no longer wanted
  {{CMD}}ergo release ABCDEF{{RESET}}    unfinished work ready for another attempt

Done, block, and cancel accept any readable leaf state. Release accepts todo,
doing, blocked, or legacy error and returns it to unclaimed todo. It rejects done
and canceled; use a specific claim to resume closed work. Every lifecycle command
clears the claim. Repeating an established postcondition adds no state event.

Append a concise lifecycle message with `-m`:

  {{CMD}}ergo block ABCDEF -m "Waiting for an API key"{{RESET}}
  {{CMD}}ergo done ABCDEF -m "Implemented" -m "Verified with task ci"{{RESET}}

Repeated messages form paragraphs in one durable message. Messages append;
they never replace the task body. Lifecycle commands reject any piped stdin,
including an empty pipe. Use `ergo body` when the body itself must change.

Attach an existing project file:

  {{CMD}}ergo done ABCDEF --result docs/verification.md{{RESET}}
  {{CMD}}ergo release ABCDEF -m "Useful partial work" --result .scratch/attempt.md{{RESET}}

The path must be relative, inside the project, outside `.ergo/`, and already
exist. The path identifies a new result. Ergo records its SHA-256, mtime, and git
commit when available. Results append as attempt history. Distinct captions in
legacy logs remain visible. Repeat the current lifecycle command to attach late
evidence.

{{HEADER}}7. DEPENDENCIES{{RESET}}

  {{CMD}}ergo sequence TASK_A TASK_B{{RESET}}
  {{CMD}}ergo sequence TASK_A TASK_B TASK_C{{RESET}}
  {{CMD}}ergo unsequence TASK_A TASK_B{{RESET}}
  {{CMD}}ergo unsequence TASK_A TASK_B TASK_C{{RESET}}

Sequence means A, then B, then C: each later task depends on the preceding task.
Unsequence removes those same adjacent edges. Existing or absent edges are
successful no-ops. A failed chain writes no partial edges.

Self-dependencies, cycles, and dependencies between a container and its child
fail. A dependency on a container completes when all children are done or
canceled.

{{HEADER}}8. OUTPUT, FLAGS, AND ERRORS{{RESET}}

Ergo has one readable output mode. List is a compact tree. Show and claim are
YAML front matter plus Markdown. Creation prints generated IDs. Focused writes
print the affected ID and postcondition. Errors go to stderr and exit nonzero.

Global flags may precede a command:

  {{CMD}}ergo --agent model@host ...{{RESET}}  caller identity
  {{CMD}}ergo --dir path/to/repo ...{{RESET}}  discovery start
  {{CMD}}ergo --help{{RESET}}                  this compact manual
  {{CMD}}ergo --version{{RESET}}               build version

Inline JSON is creation input, not an output mode. The event log is JSONL.
Ergo 3 removed output `--json`; rerun without it. It also removed `--summary`;
use `-m` for a lifecycle note. There is no `set`, `reopen`, or `sequence rm`.
Errors for these forms name the replacement.

{{HEADER}}9. MAINTENANCE AND LEGACY PLANS{{RESET}}

  {{CMD}}ergo prune{{RESET}}        preview closed work that can be removed
  {{CMD}}ergo prune --yes{{RESET}}  tombstone it
  {{CMD}}ergo compact{{RESET}}      rewrite the log to current live state
  {{CMD}}ergo version{{RESET}}      print the build version
  {{CMD}}ergo quickstart{{RESET}}   print this guide

Prune targets done and canceled leaves, then containers left empty. Compact
physically removes pruned history. Reads and writes use `.ergo/lock`; automatic
claim selects and writes under the same lock, so concurrent agents cannot claim
the same task.

Ergo reads `.ergo/plans.jsonl` and legacy `.ergo/events.jsonl` without migration.
Opening never rewrites a log. Replay and compact preserve unresolved legacy error
and claimed-blocked work. An explicit claim or lifecycle command normalizes only
the task it changes. Older events and distinct result captions remain readable.
