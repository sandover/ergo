{{BOLD}}ergo{{RESET}} — fast, local task graph for AI agents. humans welcome.

  Tasks have states (todo → doing → done/blocked/error/canceled) and can depend on
  each other. A task is "ready" when all its dependencies are done. A task with
  children is an epic (no state of its own; completes when children finish).

{{HEADER}}WORKFLOW{{RESET}}

  # Create a task (body is piped on stdin)
  printf '%s\n' '## Goal' '- Add rate limiting' | ergo --json new task '{"title":"Rate limiter"}'
  → {"id":"ABCDEF","state":"todo",...}

  # Claim next ready task
  ergo --json claim --agent sonnet@myhost
  → {"id":"ABCDEF","state":"doing","claim":"sonnet@myhost",...}

  # Mark done with result
  ergo --json done ABCDEF --result src/ratelimit.ts
  → {"id":"ABCDEF","state":"done",...}

{{HEADER}}COMMANDS{{RESET}}
  new task [json]                         create one task (stdin = body when piped)
  set <id> [json]                         update one task (stdin = replacement body when piped)
  show <id>                               task detail (Markdown default; --json for agents)
  list [--epic <id>] [--ready] [--all]    list tasks; default omits done/canceled
  claim [<id>] --agent <identity>         claim next ready task (or specific ID)
  done <id> [--result <path>]             mark the task done
  block <id> [--result <path>]            mark the task blocked
  cancel <id> [--result <path>]           cancel the task
  release <id> [--result <path>]          return unfinished work to todo
  sequence <A> <B> [<C>...]               enforce order: A before B before C
  sequence rm <A> <B>                     remove the edge where B depends on A
  plan --file <path> [json]               create epic + children from markdown file
  init [dir]                              create .ergo/ in dir (default: .)
  where                                   print active .ergo directory path
  prune [--yes]                           remove done/canceled tasks (dry-run default)
  compact                                 rewrite log to current state
  quickstart                              complete reference manual
  version                                 print version

{{HEADER}}GLOBAL FLAGS{{RESET}}
  --json                                  structured output
  --agent <identity>                      caller identity (suggested: model@host)
  --dir <path>                            discovery start (or .ergo path)
  -h, --help                              show help
  -V, --version                           print version

{{HEADER}}TASK FIELDS (JSON for new/set){{RESET}}
  Unknown keys are rejected (with suggestions for close matches).
  `new task` requires `title`. `set` accepts any subset.

  title           task title
  epic            parent task ID ("" to unassign)
  state           todo|doing|done|blocked|canceled|error
  claim           agent identity (implies state=doing)
  result          file path to attach as a result

  Stdin: piped input becomes the task body (new) or replaces it (set).
  Empty pipe = empty/cleared body. TTY = body unchanged.

{{HEADER}}PLAN{{RESET}}
  ergo --json plan --file tasks.md '{"title":"Auth system"}'

  Markdown file: chunks separated by `---`. Each chunk = `# Title` + body.
  File order does not imply dependencies — use `sequence` afterwards.

{{HEADER}}RULES{{RESET}}
  1. Never leave a task in `doing`. Finish with `done`, `blocked`, `error`, or `canceled`.
  2. `doing` and `error` require a claim. `done`/`canceled` clear it. `blocked` may keep it.
  3. Epics cannot have state, claim, or results — only leaf tasks can.
  4. One level of nesting: a child cannot itself be an epic.
  5. Dependencies: any non-ancestor tasks; cycles are forbidden.
