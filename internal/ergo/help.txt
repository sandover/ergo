{{BOLD}}ergo{{RESET}} — a fast, local task graph for coding agents.

  Tasks form a graph. Ready = todo with every dependency complete.

{{HEADER}}USAGE{{RESET}}
  ergo [global flags] <command> [args]

{{HEADER}}AGENT LOOP{{RESET}}

  ergo --json claim --agent model@host
  → {"id":"ABCDEF","state":"doing","next_commands":{...}}

  No ready → {"status":"no_ready","message":"No ready ergo tasks."}

  ergo --json done ABCDEF --result src/feature.go
  → {"kind":"done","id":"ABCDEF","updated_fields":["claim","result","state"],"state":"done"}

  Exit by intent:

    done       objective satisfied
    block      identified impediment needs action
    cancel     objective no longer needed
    release    unfinished work ready to retry

{{HEADER}}COMMANDS{{RESET}}
  init [dir]                                      create .ergo/ (default: current)
  new task [json]                                 create task; stdin is body
  plan --file <path> [json]                       create container from markdown
  show <id>                                       show task or container
  list [--epic <id>] [--ready | --all]            list work; omits done/canceled by default
  claim [<id>] --agent <identity>                 claim oldest ready or chosen task
  done <id> [--result <path>] [--summary <text>]  mark objective satisfied
  block <id> [--result <path>] [--summary <text>] record an impediment
  cancel <id> [--result <path>] [--summary <text>] stop unwanted work
  release <id> [--result <path>] [--summary <text>] return unfinished work to todo
  title <id> <title>                              replace task or container title
  body <id>                                       replace body with literal piped stdin
  move <id> <container-id>                        move leaf into container
  move <id> --root                                move leaf to root
  sequence <A> <B> [<C>...]                       require A before B before C
  sequence rm <A> <B>                             remove the edge where B depends on A
  where                                           print the active .ergo directory
  prune [--yes]                                   tombstone closed work; dry-run by default
  compact                                         rewrite the log to current live state
  quickstart                                      show the complete reference manual
  version                                         print the version

{{HEADER}}GLOBAL FLAGS{{RESET}}
  --json                                           emit one JSON value
  --agent <identity>                               identity; prefer model@host
  --dir <path>                                     discovery start or existing .ergo
  -h, --help                                       show help
  -V, --version                                    print the version

{{HEADER}}CREATE INPUT{{RESET}}
  `new task` accepts one JSON object. Unknown fields fail; `title` is required.

  title     nonblank title
  epic      destination container ID
  state     todo|doing|blocked|done|canceled
  claim     agent identity; implies doing
  result    project-relative file

  `epic` is the compatibility name for a container. A clean root todo becomes
  one on its first child. Piped stdin is the body; an empty pipe
  creates an empty body. Keep multiline Markdown out of JSON and shell arguments.

  `plan` JSON requires `title`. Its markdown file contains `# Title` chunks
  separated by a line that is exactly `---`. File order creates no dependencies.

{{HEADER}}MUTATION INPUT{{RESET}}
  Piped lifecycle stdin replaces the body in the same atomic change. TTY stdin
  leaves it unchanged.

  `body` requires piped stdin; an empty pipe clears it. `--summary` requires
  `--result`. Results must exist inside the project and outside .ergo/.

{{HEADER}}RULES{{RESET}}
  1. A claim exists exactly while state is doing; lifecycle exits clear it.
  2. Claim without an ID selects oldest ready. Specific claim may resume todo,
     blocked, done, canceled, or legacy error despite unmet dependencies.
  3. Release returns active or legacy-error work to todo. Specific claim resumes
     done or canceled work.
  4. Containers cannot be claimed, moved, nested, or hold results.
  5. Dependencies cannot be self, cyclic, or between container and child.
  6. Replay and compact preserve unresolved legacy state and ownership.

  Success exits 0; failure exits nonzero with an actionable stderr message.
  There is no `set` or `reopen`. Run `ergo quickstart` for examples.
