{{BOLD}}ergo{{RESET}} — fast, local task graph for AI agents. humans welcome.

Append-only task graph stored in `.ergo/` as JSONL. Tasks can depend on each other.
A task with children is a container. Leaf tasks carry state, claim, and results.

{{HEADER}}WORKFLOW{{RESET}}

  printf '%s\n' '## Goal' '- Add rate limiting' | ergo --json new task '{"title":"Rate limiter"}'
  → {"id":"ABCDEF","state":"todo",...}

  ergo --json claim --agent sonnet@myhost
  → {"id":"ABCDEF","state":"doing","claim":"sonnet@myhost",...}

  ergo --json set ABCDEF '{"state":"done","result":"src/ratelimit.ts"}'
  → {"id":"ABCDEF","state":"done",...}

{{HEADER}}COMMANDS{{RESET}}
  init [dir]                              create .ergo/ in dir (default: .)
  new task [json]                         create one task (stdin = body when piped)
  set <id> [json]                         update one task (stdin = replacement body when piped)
  plan --file <path> [json]               create one container and child tasks from markdown
  show <id> [--short]                     show one task (Markdown default; --json for agents)
  list [--epic <id>] [--ready] [--all]    list tasks; default omits done/canceled
  claim [<id>] --agent <identity>         claim specific task or oldest ready task
  sequence <A> <B> [<C>...]               enforce order: A before B before C
  sequence rm <A> <B>                     remove the edge where B depends on A
  prune [--yes]                           remove done/canceled tasks (dry-run default)
  compact                                 rewrite log to current state
  where                                   print active .ergo directory path
  quickstart                              complete reference manual
  version                                 print version

{{HEADER}}GLOBAL FLAGS{{RESET}}
  --dir <path>                            discovery start (or .ergo path)
  --agent <identity>                      caller identity (suggested: model@host)
  --json                                  JSON output
  -h, --help                              show help
  -V, --version                           print version

{{HEADER}}JSON ARGUMENTS FOR new/set{{RESET}}
  Unknown keys are rejected (with suggestions for close matches).
  `new task` requires `title`. `set` accepts any subset.

  title           task title
  epic            parent task ID ("" to unassign)
  state           todo|doing|done|blocked|canceled|error
  claim           agent identity
  result          file path to attach as a result

Stdin behavior:
  - If stdin is piped to `new task`, it becomes the task body.
  - If stdin is piped to `set`, it replaces the task body.
  - Empty piped stdin is allowed: it creates an empty body on `new task` and clears the body on `set`.
  - When stdin is a TTY, the body is left unchanged unless you pipe one in.

{{HEADER}}PLAN FILE FORMAT{{RESET}}
  ergo --json plan --file tasks.md '{"title":"Auth system"}'

  tasks.md:
    # Schema
    Create the tables and indexes.
    ---
    # Endpoints
    Add signup and login handlers.

  - Split chunks on a line that is exactly `---`.
  - Each chunk must start with `# Title`.
  - The heading becomes the child title; the rest of the chunk becomes the body.
  - The command JSON must include the top-level container `title`.
  - File order does not imply dependencies. Use `sequence` explicitly.

{{HEADER}}RULES{{RESET}}
  1. Never leave a task in `doing`. Finish with `done`, `blocked`, `error`, or `canceled`.
  2. Claim and `doing` move together. Finishing (`done` or `canceled`) clears the claim.
  3. Containers cannot hold state, claim, or results. Only leaf tasks can.
  4. One level of nesting only: a child cannot itself become a container.
  5. Dependencies are allowed between any two non-ancestor tasks. Cycles are forbidden.
