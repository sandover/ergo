{{BOLD}}ergo{{RESET}} — task orchestration for AI agents. humans welcome.

Append-only task graph stored in .ergo/ (JSONL). Containers are tasks with children.
Tasks have states (todo→doing→done). IDs are 6-char uppercase (e.g. OFKSTE).
Input: pipe JSON or use flags. Output: JSON objects with --json.

{{HEADER}}EXAMPLE: FULL LIFECYCLE{{RESET}}

  printf '%s' '{"title":"Add login"}' | ergo --json new task
  → {"id":"ABCDEF","state":"todo",...}

  ergo --json claim ABCDEF --agent sonnet@myhost
  → {"id":"ABCDEF","state":"doing","claim":"sonnet@myhost",...}

  printf '%s' '{"state":"done"}' | ergo --json set ABCDEF
  → {"id":"ABCDEF","state":"done",...}

For multi-line Markdown bodies, use --body-stdin (stdin is the raw text):
  printf '%s\n' '## Goal' '- Implement login' | ergo --json new task --body-stdin --title "Add login"

{{HEADER}}COMMANDS{{RESET}}
  init [dir]                              create .ergo/ in dir (default: .)
  new task                                create task (JSON stdin; or --body-stdin)
                                          include tasks:[...] to create container+children+deps
  set <id>                                update task (JSON stdin; or --body-stdin)
  show <id>                               show task (Markdown default; --json for agents)
  list [--epic <id>] [--ready] [--containers] [--all]
                                          list active work (--ready = unblocked todo tasks)
  claim [<id>] --agent <identity> [--epic <id>]
                                          claim oldest ready task (or by ID); sets to doing
  sequence <A> <B> [<C>...]               enforce order: A before B before C
  sequence rm <A> <B>                     remove edge where B depends on A
  where                                   print active .ergo directory path
  prune [--yes] [--json]                  remove done/canceled tasks (dry-run default)
  compact                                 collapse log to current state
  quickstart                              complete reference manual

{{HEADER}}GLOBAL FLAGS{{RESET}}
  --dir <path>                            discovery start (or .ergo path)
  --agent <identity>                      identity for claim and implicit set
  --json                                  JSON output (required for agents)
  --quiet, -q                             suppress output
  --verbose, -v                           verbose output
  -h, --help                              show help
  -V, --version                           print version

{{HEADER}}FLAGS FOR new/set{{RESET}}
  --body-stdin                            read body from stdin (raw text, non-empty)
  --title <text>                          set title (required for new with --body-stdin)
  --body <text>                           inline body text (mutually exclusive with --body-stdin)
  --epic <id>                             assign/reassign to container ("" to unassign)
  --state <state>                         todo|doing|done|blocked|canceled|error
  --claim <agent>                         set claim
  --result-path <path>                    attach result file (requires --result-summary)
  --result-summary <text>                 one-line result summary (requires --result-path)
  When stdin is a TTY (not piped), new/set can be driven entirely by flags.

{{HEADER}}JSON INPUT FIELDS (all strings){{RESET}}
  Unknown keys are rejected (with suggestions for close matches).
  Containers cannot have: state, claim, results (these are leaf-task only)
  title           required for new, optional for set
  body            optional (for multi-line, prefer --body-stdin over JSON escaping)
  epic            container ID or "" to unassign
  state           todo|doing|done|blocked|canceled|error
                  doing/error require claim; todo/done/canceled clear claim
  claim           agent ID or "" to unclaim
                  claim implies state=doing unless state is also set
  result_path     file path (requires result_summary)
  result_summary  one-liner (requires result_path)
  tasks           array of {title, body?, after?} for bulk creation (see quickstart §4d)

{{HEADER}}RULES{{RESET}}
  Never leave a task in doing — always finish with done, blocked, error, or canceled.
  Do not hand-escape Markdown into JSON. Use --body-stdin for multi-line bodies.
