{{BOLD}}ergo{{RESET}} — a fast, local task graph for coding agents.

Tasks are stateful and can depend on one another. A todo task is ready when all
its dependencies are complete. A task with children is a container; it has no
state of its own and completes when all its children are done or canceled.

{{HEADER}}USAGE{{RESET}}
  ergo [global flags] <command> [args]

{{HEADER}}WORKFLOW{{RESET}}
  {{CMD}}ergo init{{RESET}}
  {{CMD}}ergo new task '{"title":"Add login"}'  # returns task ID ABCDEF{{RESET}}
  {{CMD}}ergo list --ready                        # shows available work{{RESET}}
  {{CMD}}ergo claim ABCDEF --agent model@host{{RESET}}
  {{CMD}}ergo done ABCDEF -m "Implemented and verified"{{RESET}}

{{HEADER}}COMMANDS{{RESET}}
  init [dir]                                 create .ergo/; print its path
  new task [json]                            create a leaf; stdin is its body
  plan --file <path> [json]                  create a container and children
  list [--epic <id>] [--ready | --all]       list work
  show <id>                                  show a leaf or container
  claim [<id>] --agent <identity>            claim oldest ready or chosen work
  done <id> [-m <text>] [--result <path>]    objective satisfied
  block <id> [-m <text>] [--result <path>]   impediment requires action
  cancel <id> [-m <text>] [--result <path>]  objective no longer wanted
  release <id> [-m <text>] [--result <path>] unfinished work ready to retry
  title <id> <title>                         replace a title
  body <id>                                  replace a body from piped stdin
  move <id> <container-id>                   move a leaf into a container
  move <id> --root                           move a leaf to root
  sequence <A> <B> [<C>...]                  require A before B before C
  unsequence <A> <B> [<C>...]                remove that order
  where                                      print the active .ergo path
  prune [--yes]                              preview or apply closed-work pruning
  compact                                    rewrite the log to live state
  quickstart                                 print the full guide
  version                                    print the build version

{{HEADER}}GLOBAL FLAGS{{RESET}}
  --agent <identity>  caller identity; prefer model@host
  --dir <path>        start discovery at a path or existing .ergo directory
  -h, --help          print this help
  -V, --version       print the build version

{{HEADER}}CREATE AND EDIT{{RESET}}
`new task` requires an inline JSON `title`. Optional fields are `epic`, `state`,
`claim`, and `result`; unknown fields fail. `epic` is the container ID. Claim
implies doing. Keep multiline Markdown out of JSON: pipe it into `new task`.

  {{CMD}}printf '%s\n' '## Goal' '- Add login' | ergo new task '{"title":"Login"}'{{RESET}}
  {{CMD}}ergo title ABCDEF "Add password login"{{RESET}}
  {{CMD}}printf '%s\n' 'New body' | ergo body ABCDEF{{RESET}}
  {{CMD}}ergo move ABCDEF OFKSTE{{RESET}}
  {{CMD}}ergo move ABCDEF --root{{RESET}}

`body` is the only command that changes an existing body. It requires a pipe;
an empty pipe clears the body. Lifecycle commands reject piped stdin.

`plan` requires `{"title":"Container title"}`. Its file contains `# Title`
chunks separated by a line that is exactly `---`. File order adds no dependency.

{{HEADER}}LIFECYCLE{{RESET}}
Leaves are todo, doing, blocked, done, or canceled. Doing work has a claim; every
lifecycle command clears it. A specific claim can resume blocked, done, or
canceled work even when its dependencies are incomplete.

Done, block, and cancel accept any readable leaf state. Release returns
unfinished work to todo; it rejects done and canceled. Containers have no state,
claim, message, or result.

`-m` appends a lifecycle message; repeat it to join paragraphs. `--result`
attaches an existing project-relative file outside `.ergo/`. New results are
identified by path.

  {{CMD}}ergo block ABCDEF -m "Waiting for an API key"{{RESET}}
  {{CMD}}ergo done ABCDEF -m "Verified with task ci" --result docs/spec.md{{RESET}}

{{HEADER}}GRAPH AND MAINTENANCE{{RESET}}
`sequence A B` makes B depend on A; longer forms connect adjacent pairs.
`unsequence` removes the same edges. Self-dependencies, cycles, and edges between
a container and its child fail. A container cannot be claimed, moved, nested,
or hold results. A move destination must be a root container or a clean root
todo task, which becomes a container on its first child.

List omits done and canceled by default. `--ready` shows claimable leaves;
`--all` includes closed work; they conflict. `--epic` selects one container.

Prune is a dry-run until `--yes`; compact removes pruned history.
